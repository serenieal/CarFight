## 1. 현재 목표

이 폴더는 CarFight의 Dedicated Server 전환 작업을 관리한다.

현재 단계의 목표는 멀티플레이 전체 완성이 아니다.

현재 목표는 다음 하나로 제한한다.

```text
현재 이미 구현되어 있는 기능만 Dedicated Server 기준으로 안정화한다.
```

따라서 현재 단계에서는 아래 작업을 하지 않는다.

```text
- 무기 시스템 신규 구현
- 체력 시스템 신규 구현
- 대미지 시스템 신규 구현
- 스코어 시스템 신규 구현
- 킬/데스 시스템 신규 구현
- 리스폰 시스템 신규 구현
- 로비 / 세션 / 매치메이킹 구현
```

위 기능들은 삭제된 것이 아니라, 현재 구현 기능 안정화가 끝난 뒤 확장 단계에서 다시 계획한다.

## 2. 문서 목록

### 2.1 `Scope.md`

현재 작업 범위를 고정하는 문서다.

가장 먼저 읽어야 한다.

포함 내용:

```text
- 현재 구현된 기능만 안정화한다는 원칙
- 미구현 기능의 현재 범위 제외
- 확장 단계 진입 조건
```

---

### 2.2 `MPPlan.md`

Dedicated Server 전환 상위 계획서다.

포함 내용:

```text
- 현재 안정화 단계의 목표
- 서버/클라이언트 책임 원칙
- 현재 포함 범위와 제외 범위
- 안정화 완료 기준
```

---

### 2.3 `FeatureList.md`

현재 구현된 기능을 목록화하는 문서다.

사용 목적:

```text
- 실제 프로젝트에 존재하는 기능만 기록
- 기능별 서버/클라이언트 책임 분류
- 현재 안정화 대상 선정
```

---

### 2.4 `NetAudit.md`

현재 구현된 기능을 멀티플레이 관점에서 점검하는 체크리스트다.

사용 목적:

```text
- Dedicated Server에서 실행되면 안 되는 코드 찾기
- 소유자 전용 기능과 서버 권한 기능 분리
- UI, 카메라, 사운드, 이펙트의 서버 의존 제거
- Actor 복제, 소유권, Spawn 위치 점검
```

---

### 2.5 `DSMinRun.md`

Dedicated Server 최소 실행선 문서다.

사용 목적:

```text
- Dedicated Server 실행 기준 정의
- 클라이언트 2개 접속 기준 정의
- Pawn 또는 차량 소유권 확인 절차 정의
- 서버/클라이언트 로그 확인 기준 정의
```

---

### 2.6 `PhaseTasks.md`

현재 안정화 단계를 실제 작업 단위로 쪼갠 문서다.

사용 목적:

```text
- 현재 구현 기능 목록화
- 네트워크 위험 요소 점검
- Dedicated Server 최소 실행선 통과
- 현재 구현 기능 안정화 완료 기준 관리
```

## 3. 권장 읽기 순서

처음 확인할 때는 아래 순서로 읽는다.

```text
1. README.md
2. Scope.md
3. MPPlan.md
4. FeatureList.md
5. NetAudit.md
6. DSMinRun.md
7. PhaseTasks.md
```

실제 작업 중에는 아래 순서로 사용한다.

```text
1. Scope.md로 현재 범위 확인
2. FeatureList.md에 실제 구현 기능만 기록
3. NetAudit.md로 현재 구현 기능만 점검
4. DSMinRun.md로 Dedicated Server 최소 실행선 테스트
5. PhaseTasks.md의 현재 안정화 작업만 진행
6. 범위 변경이 필요하면 Scope.md와 MPPlan.md를 먼저 갱신
```

---

## 4. 현재 기준 원칙

```text
- 새 시스템을 만들지 않는다.
- 현재 구현된 기능만 안정화한다.
- 서버는 실제 상태와 판정을 담당한다.
- 클라이언트는 입력, UI, 카메라, 사운드, 이펙트를 담당한다.
- Dedicated Server에서 로컬 플레이어, HUD, 카메라에 의존하지 않는다.
- 안정화 완료 후 무기, 체력, 스코어 같은 확장 시스템을 다시 계획한다.
```
