# 현재 작업 범위 고정 문서

- 문서 버전: 0.1.0
- 작성일: 2026-05-22
- 대상 프로젝트: CarFight
- 상위 문서: `MPPlan.md`

---

## 1. 목적

이 문서는 CarFight Dedicated Server 전환 작업의 현재 범위를 명확히 고정한다.

현재 작업의 핵심은 다음이다.

```text
새 시스템을 추가하지 않는다.
지금 이미 구현되어 있는 기능만 대상으로 한다.
그 구현물을 Dedicated Server 기준으로 안정화한다.
안정화가 끝난 뒤 무기, 체력, 스코어 같은 확장 시스템으로 넘어간다.
```

---

## 2. 현재 범위 결정

현재 단계의 작업 범위는 다음과 같이 제한한다.

```text
현재 구현된 기능만 점검한다.
현재 구현된 기능만 수정한다.
현재 구현된 기능만 Dedicated Server에서 안정화한다.
```

아직 구현되지 않은 기능은 현재 작업 범위에 포함하지 않는다.

---

## 3. 현재 포함 범위

실제 프로젝트에 이미 구현되어 있는 경우에만 아래 기능을 점검한다.

```text
- 차량 이동
- 차량 입력
- 차량 소유권
- 차량 카메라
- HUD / UI
- 부스터
- 드리프트
- 차량 충돌
- 차량 스폰
- 차량 선택
- 사운드
- 이펙트
- 맵 오브젝트
- 디버그 기능
```

중요:

```text
위 목록에 있더라도 실제 프로젝트에 아직 구현되어 있지 않으면 작업하지 않는다.
```

---

## 4. 현재 제외 범위

아래 기능은 현재 작업에서 제외한다.

```text
- 무기 시스템
- 체력 시스템
- 대미지 시스템
- 스코어 시스템
- 킬/데스 시스템
- 리스폰 시스템
- 킬 로그
- 스코어보드
- 매치 승패
- 로비
- Steam 세션
- 매치메이킹
- 서버 브라우저
- 서버 배포 자동화
```

이 기능들은 삭제된 것이 아니다.

현재 안정화 단계에서 제외하고, 이후 확장 단계에서 다시 계획한다.

---

## 5. 현재 단계 완료 기준

현재 단계는 아래 조건을 만족하면 완료로 본다.

```text
1. 현재 구현된 기능 목록이 정리되어 있다.
2. 현재 구현된 기능의 서버/클라이언트 책임이 분류되어 있다.
3. Dedicated Server에서 클라이언트 2개가 접속할 수 있다.
4. 각 클라이언트가 자기 Pawn 또는 차량을 올바르게 소유한다.
5. 현재 구현된 이동 또는 조작 기능이 소유자 기준으로 동작한다.
6. 다른 클라이언트에서 관찰되어야 하는 현재 구현 기능이 정상적으로 보인다.
7. Dedicated Server에서 HUD, 카메라, 로컬 사운드 때문에 오류가 나지 않는다.
8. 아직 구현되지 않은 기능을 새로 만들지 않고 안정화 작업을 마친다.
```

---

## 6. 확장 단계로 넘어가는 조건

무기, 체력, 스코어 같은 신규 시스템은 아래 조건을 만족한 뒤 다시 계획한다.

```text
1. 현재 구현 기능 안정화 완료
2. Dedicated Server 최소 실행선 통과
3. 현재 구현 기능의 위험 코드 목록 정리 완료
4. 서버/클라이언트 책임 분리 완료 또는 보류 사유 기록
5. 다음 확장 대상 하나를 선택
```

확장 단계는 한 번에 여러 시스템을 만들지 않는다.

예시:

```text
1차 확장 후보:
- 무기 발사/복제만 구현

2차 확장 후보:
- 체력/대미지 구현

3차 확장 후보:
- 스코어/킬/데스/리스폰 구현
```

---

## 7. 문서 해석 규칙

기존 문서에 무기, 체력, 스코어, 리스폰 같은 항목이 등장하더라도 현재 단계에서는 아래처럼 해석한다.

```text
- 현재 구현되어 있지 않으면 작업하지 않는다.
- 현재 구현 기능 안정화 이후의 확장 후보로만 본다.
- 지금 당장 C++ 클래스나 Blueprint를 새로 만들지 않는다.
- 현재 작업의 완료 기준에 포함하지 않는다.
```

---

## 8. 현재 우선 작업

현재 우선 작업은 다음과 같다.

```text
1. 실제 구현 기능 목록 확인
2. 현재 구현 기능만 FeatureList.md에 반영
3. 현재 구현 기능만 NetAudit.md로 점검
4. Dedicated Server 최소 실행선 테스트
5. 현재 구현 기능에서 발생한 문제만 수정
6. 안정화 완료 후 다음 확장 시스템 재논의
```
