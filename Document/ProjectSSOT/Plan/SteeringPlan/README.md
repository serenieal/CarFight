# SteeringPlan README

## 목적
`Document/ProjectSSOT/Plan/SteeringPlan/`은 CarFight 차량 조향 입력 변경안을 정리하는 폴더다.

이번 조향 변경의 핵심은 아래다.

- 스틱 기울기 정도는 가속 / 브레이크 강도에 사용한다.
- 스틱이 기울어진 방향은 조향 목표에 사용한다.
- 스틱을 덜 기울였다고 바퀴도 덜 꺾이는 구조는 사용하지 않는다.
- 실제 바퀴 조향값은 목표 조향값으로 즉시 점프하지 않고 제한 속도로 따라간다.
- 스틱 중립 시 바퀴는 즉시 0으로 가지 않고, 차량 속도에 비례해서 천천히 중립 복귀한다.
- 고속에서 급조향하면 차량이 불안정해질 수 있으며, 이는 자연스러운 리스크로 허용한다.

## 문서 목록

### 1. `CF_SteerSpec.md`
- 조향 변경의 최종 명세서
- 스틱 크기/방향 분리 규칙
- 조향 목표값 / 실제 조향값 / 중립 복귀 규칙 정의

### 2. `CF_SteerDesign.md`
- C++ 구현 구조 설계
- `ACFVehiclePawn` 기준 적용 위치
- 조향 속도 제한과 속도 기반 중립 복귀 계산 방식 정의

### 3. `CF_SteerTasks.md`
- 실제 구현 작업 순서
- 헤더/CPP 수정 항목
- 튜닝 변수와 디버그 항목 추가 순서

### 4. `CF_SteerTest.md`
- PIE 테스트 케이스
- 조향/가속 분리 검증
- 조향 속도 제한 및 중립 복귀 검증

### 5. `CF_SteerRisk.md`
- 문제점 분석
- 수정 후 생길 수 있는 리스크
- 허용할 리스크와 막아야 할 리스크 구분

## 현재 결정 요약
현재 채택된 방향은 아래 한 줄로 요약된다.

**스틱 크기는 가감속, 스틱 방향은 목표 조향, 실제 조향은 제한 속도로 목표를 따라가며 중립 복귀는 차량 속도에 비례한다.**
