# 프로젝트 작업 규칙 v1.0

## 기본 언어
- 항상 기본적으로 한국어로 답변한다.
- 사용자가 다른 언어를 명시적으로 요청한 경우에만 해당 언어를 사용한다.

## UE 작업 자동 적용 규칙
- 사용자가 별도로 `$ue-ai-collab`를 직접 호출하지 않아도, 프롬프트에 아래 성격이 보이면 자동으로 `Document/SSOT/Codex_SSOT/skills/ue-ai-collab` 스킬 규칙을 우선 적용한다.
- 적용 트리거 예시:
  - 언리얼, Unreal, UE, UE5
  - 블루프린트, Blueprint, BP, AnimBP
  - Pawn, Character, GameMode, PlayerController, Component
  - 차량, Vehicle, Chaos Vehicle, Wheel, Suspension
  - UE 에디터 메뉴, 세부 정보 패널, 클래스 기본값, 이벤트 그래프
  - BP/C++ 전환, 리팩터링, 부모 클래스 교체, 컴포넌트 구조 재정리
- 사용자가 명시적으로 다른 작업 방식을 요구하지 않는 한, UE 관련 요청은 위 스킬을 자동 적용한 것으로 간주한다.
- 사용자가 자연어로만 요청해도 된다. 스킬 호출 명령어를 매번 요구하지 않는다.
- 다른 PC에서는 `Document/SSOT/Codex_SSOT/Install-CodexSSOT.ps1`를 실행해 로컬 Codex 스킬 폴더로 설치한다.
- UE C++ 구조화, BP->C++ 전환, BP 허용/금지 판정, 예외 처리, 판정 로그 작성이 포함된 작업이면 `Document/SSOT/UE_SSOT/UE_CPP_SSOT/UE_SSOT_Split_Index_v0_1.md`를 먼저 보고 관련 분리 문서를 함께 적용한다.

## UE 응답 기본 원칙
- UE 작업은 한국어판 에디터 기준으로 설명하고, 필요 시 한국어 UI명(영문명) 형식으로 병기한다.
- 기존 BP가 과도기 자산으로 보이면 기준 구현으로 단정하지 않는다.
- 가능하면 사용자 에디터 작업과 AI 코드 작업을 분리해서 안내한다.
- 단계별 테스트, 결과 해석, 다음 단계를 함께 제시한다.
- UE C++ 관련 작업에서는 `UE_CPP_SSOT` 분리 문서의 판정 기준, 허용/금지 패턴, 프로젝트 예외 원칙을 기본 적용한다.

## 문서 우선순위
- 현재 사용자 지시
- 프로젝트 전용 SSOT
- 작업별 계획서/마이그레이션 문서
- 공용 UE SSOT
- 기타 공용 운영 문서

## 예외 처리
- UE 관련 요청이 아니면 일반 규칙으로 처리한다.
- UE 관련 요청이어도 프로젝트 전용 규칙이 더 구체적이면 프로젝트 문서를 우선한다.
