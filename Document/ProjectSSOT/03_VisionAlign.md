# CarFight — 03_VisionAlign

> 역할: CarFight의 **상위 비전 / 현재 구현 / 다음 확장 방향**을 한 문서에서 연결한다.  
> 마지막 정리(Asia/Seoul): 2026-03-20

---

## 한 줄 정의
CarFight는 **차량 전투와 물리 상호작용**을 중심으로, 다양한 차량을 하나의 공통 차량 코어 위에 올릴 수 있도록 만드는 프로젝트다.

---

## 프로젝트의 상위 목표
### 1. 차량 전투 중심 게임플레이
- 단순 주행만이 아니라 차량 간 충돌, 밀기, 제어 상실, 복구 같은 **물리 기반 상호작용**이 중요하다.
- 최종적으로는 전투 규칙, 충돌 결과, 카메라 반응, 연출까지 차량 상태와 연결되어야 한다.

### 2. 다양한 차량 대응
- 프로젝트 목표는 단일 차량 완성이 아니라 **여러 차종을 같은 게임 안에 안정적으로 올리는 것**이다.
- 차량별 감각 차이는 허용하되, 공통 코어를 먼저 확보한다.

### 3. 데이터 기반 확장
- 차량별 차이는 코드 복붙이 아니라 `UCFVehicleData` 중심의 데이터 구조로 관리한다.
- 공통 규칙은 C++ 코어에 두고, 차량별 차이는 데이터와 설정값으로 분리한다.

---

## 현재 구현 기준
- 현재 기준 플레이 차량: `BP_CFVehiclePawn`
- 현재 기준 Native Pawn: `ACFVehiclePawn`
- 현재 기준 주행 코어: `UCFVehicleDriveComp`
- 현재 기준 휠 시각 동기화 코어: `UCFWheelSyncComp`
- 현재 기준 데이터 축: `UCFVehicleData`
- 현재 기준 테스트 자산: `DA_PoliceCar`

이 기준은 **지금 실제로 굴러가는 구현 기준선**이며,
프로젝트의 상위 목표를 축소한다는 뜻이 아니다.

---

## 현재 개발 단계의 의미
지금 단계는 단순 안정화만을 위한 닫힌 단계가 아니다.

정확히는 아래와 같이 본다.
- 공통 차량 코어를 신뢰 가능한 상태로 만든다.
- 그 코어 위에 여러 차종을 얹을 수 있도록 기준을 고정한다.
- 이후 차량별 감각, 전투 반응, 카메라, 연출 확장으로 이어진다.

즉 현재 작업은 **최종 비전 이전의 임시 축소판**이 아니라,
**다차종 차량 전투 게임을 만들기 위한 기반 공사**다.

---

## 현재 문서가 따라야 할 원칙
### 1. 상위 비전을 숨기지 않는다
- 활성 문서는 현재 구현만 적더라도, 그것이 상위 목표와 어떻게 연결되는지 드러내야 한다.

### 2. 현재 기준선과 최종 목표를 구분한다
- `DA_PoliceCar`는 현재 검증 기준선이다.
- 최종 목표는 다양한 차량 대응이다.

### 3. Thin BP 원칙은 유지한다
- 판단 / 상태 / 규칙은 C++ 코어를 우선 기준으로 둔다.
- `BP_CFVehiclePawn`는 표현 / 조립 / 반응 중심으로 유지한다.

### 4. 다차종 확장을 항상 염두에 둔다
- DriveState, Wheel Sync, VehicleData는 한 차량만을 위한 구조가 아니라 공통 코어여야 한다.
- 문서도 특정 테스트 차량만을 최종 목표처럼 쓰지 않는다.
