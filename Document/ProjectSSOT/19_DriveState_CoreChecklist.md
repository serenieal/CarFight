# DriveState 공통 코어 검증 체크리스트

> 역할: 차종별 감각 튜닝 전에, **공통 코어가 최소 기준을 통과하는지** 확인하는 검증 문서  
> 적용 단계: 차량별 세부 튜닝 이전  
> 상위 방향 문서: `03_VisionAlign.md`  
> 마지막 정리(Asia/Seoul): 2026-03-20

---

## 이 문서의 목적
이 문서는 체크만 하는 문서가 아니다.

현재 목적은 아래 3가지다.
- 공통 차량 코어가 지금 단계에서 통과해야 할 최소 기준을 고정한다.
- 테스트 중 코어 문제와 차량별 튜닝 문제를 분리한다.
- 다음 단계로 넘어가도 되는지 중지선 역할을 한다.

---

## 시작 전에 확인할 것
### VehicleData 조건
- `DriveStateConfig.bUseDriveStateOverrides = false`

### Pawn 디버그 조건
- `bEnableDriveStateOnScreenDebug = true`
- `bShowDriveStateTransitionSummary = true`

### 확인할 화면 정보
- `DriveState`
- `Prev`
- `Candidate`
- `Hold`
- `Speed`
- `Forward`
- `Grounded`

### 이번 테스트의 원칙
- 차종별 감각 튜닝 금지
- 공통 기본값으로만 검증
- 이상이 생기면 먼저 **코어 문제인지** 확인
- 결과는 반드시 문서에 남김

---

## 테스트 순서
아래 순서대로만 테스트한다.

1. 전진 진입 / 유지
2. 입력 해제 후 Coasting
3. 브레이크 / Idle 복귀
4. 후진 진입 / 유지
5. 전진↔후진 경계
6. 작은 요철 Grounded 유지
7. 큰 점프/낙하 Airborne 진입
8. 착지 후 상태 복귀
9. 디버그 표시 가독성

한 항목이 실패하면 원인을 적고 다음으로 넘어가되,
같은 원인이 반복되면 테스트를 멈추고 수정 후보를 적는다.

## 항목별 통과 기준
### A. 전진 입력 진입
#### 테스트
- 정지 상태에서 전진 입력

#### 기대 결과
- `Idle -> Accelerating`
- 차가 즉시 앞으로 움직이기 시작함
- `Changed=True`가 정상적으로 보임

#### 실패 예시
- 입력했는데 `Idle` 유지
- `Braking`으로 잘못 감
- 전진은 하는데 상태가 안 바뀜

---

### B. 전진 유지
#### 테스트
- 전진 입력 유지

#### 기대 결과
- `Accelerating` 유지
- 상태가 불필요하게 흔들리지 않음

#### 실패 예시
- `Accelerating ↔ Coasting` 반복
- `Braking`이 섞임

---

### C. 입력 해제 후 Coasting
#### 테스트
- 전진 중 입력 해제

#### 기대 결과
- `Accelerating -> Coasting`
- 차량이 관성으로 조금 더 감

#### 실패 예시
- 바로 `Idle`
- 이유 없이 `Braking`

---

### D. 브레이크 진입
#### 테스트
- 전진 중 브레이크 입력

#### 기대 결과
- `Accelerating / Coasting -> Braking`
- 감속이 자연스럽게 시작됨

#### 실패 예시
- 브레이크 입력인데 `Coasting` 유지
- 바로 `Reversing`으로 감

---

### E. 정지 판정(Idle)
#### 테스트
- 브레이크 후 거의 멈춤

#### 기대 결과
- 충분히 느려지면 `Idle` 진입
- 정지 근처에서 `Idle ↔ Coasting`이 과하게 튀지 않음

#### 실패 예시
- 멈췄는데 계속 `Braking`
- 작은 흔들림으로 상태가 반복 전환됨

---

### F. 후진 진입
#### 테스트
- 정지 상태 또는 거의 멈춘 상태에서 후진 입력

#### 기대 결과
- `Idle / Braking -> Reversing`
- 후진으로 명확하게 진입

#### 실패 예시
- 후진 입력인데 계속 `Braking`
- 후진은 하는데 상태가 `Idle` 또는 `Coasting`

---

### G. 후진 유지
#### 테스트
- 후진 입력 유지

#### 기대 결과
- `Reversing` 유지
- 상태가 불필요하게 흔들리지 않음

#### 실패 예시
- `Reversing ↔ Braking` 반복
- 후진 중 `Accelerating` 표시

---

### H. 전진↔후진 경계 안정성
#### 테스트
- 전진 중 반대 방향 입력
- 후진 중 전진 입력

#### 기대 결과
- 반대 방향 입력은 우선 `Braking`
- 충분히 감속/정지 후 반대 방향 상태로 전환

#### 실패 예시
- 전진 중 바로 `Reversing`
- 후진 중 바로 `Accelerating`
- 경계에서 상태가 떨림

---

### I. 작은 요철에서 Grounded 유지
#### 테스트
- 작은 턱, 요철, 미세 점프 통과

#### 기대 결과
- 대부분 `Grounded=True` 유지
- 작은 흔들림만으로 `Airborne` 진입하지 않음

#### 실패 예시
- 작은 요철마다 `Airborne`
- `Grounded`가 심하게 흔들림

---

### J. 큰 점프/낙하에서 Airborne 진입
#### 테스트
- 분명한 점프, 낙하, 공중 체류 상황 만들기

#### 기대 결과
- `Airborne` 진입
- 공중 상태가 잠깐이라도 정상 표시됨

#### 실패 예시
- 점프했는데 끝까지 `Grounded`
- 공중인데 `Coasting`만 유지

---

### K. 착지 후 상태 복귀
#### 테스트
- Airborne 후 착지

#### 기대 결과
- 착지 후 적절한 상태로 복귀
- 상태가 한 프레임씩 과하게 튀지 않음

#### 실패 예시
- 착지 직후 `Airborne ↔ Coasting` 반복
- 계속 `Airborne` 유지

---

### L. 디버그 표시 가독성
#### 테스트
- 화면 디버그를 보면서 주행

#### 기대 결과
- 현재 상태, 후보 상태, Hold, Grounded를 읽을 수 있음
- 현재 어떤 문제인지 추적 가능한 수준이어야 함

#### 실패 예시
- 정보가 너무 자주 바뀌어 못 읽음
- 필요한 값이 없음

---

## 1차 통과 기준
아래 조건을 만족하면 공통 코어 1차 통과로 본다.

- 전진 / 브레이크 / 정지 / 후진 흐름이 자연스럽다.
- 반대 방향 입력이 즉시 반대 방향 가속으로 가지 않는다.
- 작은 요철에서 `Airborne` 오탐이 심하지 않다.
- 큰 점프/낙하에서는 `Airborne`이 확인된다.
- 상태가 경계에서 심하게 튀지 않는다.
