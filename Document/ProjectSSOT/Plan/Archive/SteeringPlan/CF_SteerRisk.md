# CF_SteerRisk

## 문서 목적
본 문서는 CarFight 차량 조향 변경안의 문제점, 해결 방향, 수정 후 리스크를 정리한다.

---

## 1. 확인된 문제점
현재 게임패드 신규 조작에서 조향은 스틱 X값에 직접 비례한다.

문제 현상:

- 스틱을 약간 움직이면 바퀴도 약간만 꺾인다.
- 스틱을 바깥쪽 끝까지 움직여야 바퀴도 끝까지 꺾인다.

이 현상은 다음 이유로 목표 조작감과 맞지 않는다.

- 스틱 크기와 바퀴 조향량이 묶여 있다.
- 살살 가속하면서 최대 조향을 하는 상황이 어렵다.
- 실제 자동차의 핸들/페달 관계와 다르다.

---

## 2. 근본 원인
현재 구조는 아래와 같다.

```text
Steering = RawStickX
Throttle / Brake = StickMagnitude
```

즉, 스틱을 덜 기울이면 `RawStickX`도 작아진다.
결과적으로 가속 강도를 줄이려고 스틱을 살짝만 기울이면 조향량도 같이 줄어든다.

---

## 3. 해결 방향
해결 방향은 입력 벡터를 크기와 방향으로 분해하는 것이다.

```text
StickMagnitude -> Throttle / Brake
StickDirection -> TargetSteering
```

그리고 실제 조향은 목표 조향값을 제한 속도로 따라가게 한다.

```text
CurrentSteering -> TargetSteering
```

---

## 4. 허용할 리스크
아래 리스크는 의도된 조작 결과로 허용한다.

### 4-1. 고속 급조향으로 인한 차량 불안정
실제 차량도 고속에서 핸들을 급하게 돌리면 불안정해진다.
CarFight에서도 이 결과는 자연스러운 리스크로 본다.

따라서 1차 구현에서는 고속 조향 입력 자체를 약화하지 않는다.

### 4-2. 이전보다 적극적인 조향 체감
스틱 방향 기반 조향으로 바뀌면 스틱을 조금만 기울여도 목표 조향이 크게 잡힐 수 있다.
이는 이번 변경의 의도다.

---

## 5. 막아야 할 리스크
아래 리스크는 구현에서 반드시 막아야 한다.

### 5-1. 입력 노이즈 증폭
정규화 방식은 작은 입력도 큰 방향값으로 바꿀 수 있다.

예:

```text
RawInput = (0.02, 0.0)
NormalizedDirection = (1.0, 0.0)
```

대응:

- `SteeringDirectionMinMagnitude`를 둔다.
- 최소 크기보다 작은 입력은 조향 의도 없음으로 본다.

### 5-2. 바퀴 조향값 순간이동
목표 조향을 바로 적용하면 바퀴가 순간적으로 반대쪽 끝까지 이동할 수 있다.

대응:

- `CurrentSteeringInput`과 `TargetSteeringInput`을 분리한다.
- `FInterpConstantTo` 같은 제한 속도 보간을 사용한다.

### 5-3. 스틱 중립 시 즉시 0 리셋
스틱을 놓았을 때 조향값이 바로 0이 되면 실제 자동차 느낌과 다르다.

대응:

- 중립 목표는 0으로 두되, 실제 조향값은 속도 기반 복귀로 이동한다.
- release 경로에서 `CurrentSteeringInput`을 즉시 0으로 만들지 않는다.

### 5-4. 입력 이벤트가 없을 때 중립 복귀가 멈춤
중립 복귀는 입력 이벤트가 없는 동안에도 계속 진행되어야 한다.

대응:

- Tick 또는 차량 입력 갱신 루프에서 중립 복귀를 계속 처리한다.

### 5-5. 기존 Runtime Input Ownership과 충돌
기존 입력 충돌 방지 로직이 release 시 조향을 0으로 리셋할 수 있다.

대응:

- `VehicleMove2D` release 시 조향을 즉시 0으로 reset하지 않는 구조를 검토한다.
- `LegacyAxis`와 `VehicleMove2D`의 조향 처리 경계를 명확히 한다.

---

## 6. 구현 전 확인 필요사항
구현 전 아래를 확인해야 한다.

- `ACFVehiclePawn`에서 Tick이 활성화되어 있는지
- release 경로에서 조향값을 즉시 0으로 리셋하는 위치
- `UCFVehicleDriveComp::ApplySteeringInput`이 별도 보간을 가지고 있는지
- 기존 키보드 조향에도 같은 보간을 적용할지 여부
