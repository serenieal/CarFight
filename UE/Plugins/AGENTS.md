이 경로 이하에서는 언리얼 플러그인 작업 규칙을 따른다.

Public/Private 경로 규칙을 반드시 준수한다.
플러그인 경계 밖으로 불필요하게 의존성이 새지 않게 한다.
기존 공개 API와 파일 구조는 말 없이 바꾸지 않는다.
무분별한 리네이밍을 금지한다.

코드 규칙:
- 변수/함수 바로 위 한 줄 주석
- 직관적인 이름 사용
- 버전 표기 포함
- changelog 포함
- migration 지침 포함

## 이성수준 기준
- 낮음: 플러그인 파일 탐색, 모듈 구조 확인, 짧은 설명, 좁은 범위 수정
- 중간: 일반 기능 구현, 로컬 수정, 소규모 리팩토링
- 높음: 공개 API 변경 검토, 다중 모듈 영향 분석, 디버깅, 구조 변경
- 플러그인 경계와 호환성 검토가 필요하면 보수적으로 높음으로 올린다.

플러그인 수정 시에는 반드시 다음을 함께 적는다.
- 어떤 모듈이 영향받는지
- 공개 API 영향 여부
- UE 프로젝트 쪽 수정이 추가로 필요한지
- 검증 방법과 체크포인트
