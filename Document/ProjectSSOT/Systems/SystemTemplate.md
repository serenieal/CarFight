# [기능명]

## 문서 목적
이 문서는 현재 프로젝트에서 `[기능명]` 기능이 실제로 어떤 일을 하는지, 그리고 그 기능이 어떤 자산/클래스/설정 구성으로 동작하는지를 기록한다.
이 문서는 미래 설계나 개선 계획이 아니라, **현재 확인된 구현 상태**를 기준으로 작성한다.

## 문서 범위
이 문서에서 말하는 `[기능명]` 기능은 아래 요소를 묶어서 본다.

- 표시/사용 자산: `[예: /Game/... ]`
- 핵심 로직 주체: `[예: AXXX / UXXX / BP_XXX ]`
- 실제 생성/호출/적용 주체: `[예: BP_XXX / GameMode / Controller / Component ]`
- 관련 설정/데이터: `[예: DataAsset / Config / InputMapping ]`

즉, 현재 기준 `[기능명]`은 **[위젯/컴포넌트/블루프린트/데이터] 하나만의 기능이 아니라, [현재 이 기능을 구성하는 묶음]**으로 본다.

## 이 기능이 현재 실제로 하는 일
현재 구현 기준 `[기능명]`의 핵심 역할은 **[이 기능의 가장 중요한 현재 역할을 한 문장으로 요약]**이다.

이 문서에서 가장 중요한 섹션이다.
여기서는 "이 기능이 존재한다" 수준이 아니라, **현재 실제로 무엇을 입력받고, 무엇을 계산/조합/판단하고, 무엇을 출력/적용/표시하는지**를 구체적으로 적는다.

현재 구현 기준으로 보면, 이 기능은 아래 정보를 처리한다.

### 1. [핵심 역할 1]
현재 이 기능은 `[무엇]`을 처리한다.

- `[현재 표시/처리/적용 항목 1]`
- `[현재 표시/처리/적용 항목 2]`
- `[현재 표시/처리/적용 항목 3]`

이 항목의 현재 의미:
- `[왜 이 항목이 필요한지]`
- `[현재 기능 안에서 이 항목이 어떤 역할을 하는지]`

### 2. [핵심 역할 2]
현재 이 기능은 `[무엇]`도 함께 처리한다.

- `[항목 1]`
- `[항목 2]`
- `[항목 3]`

이 항목의 현재 의미:
- `[실제 동작 의미]`
- `[사용자/개발자가 이 기능을 통해 읽거나 확인할 수 있는 것]`

### 3. [핵심 역할 3]
현재 구현에서 추가로 중요한 역할이 있다면 적는다.

- `[항목 1]`
- `[항목 2]`
- `[항목 3]`

중요한 이유:
- `[이 항목이 없으면 무엇을 놓치게 되는지]`

### 4. [상태 변화 / 조건 / 전이 / 예외 처리]
이 기능이 단순 정적 정보 표시가 아니라면, **상태 변화 / 조건 분기 / 전이 / 예외 처리**를 따로 적는다.

예:
- `[상태 전이 요약]`
- `[표시 조건]`
- `[실행 조건]`
- `[비활성 조건]`
- `[실패 시 동작]`

### 5. [최종 출력 / 적용 결과]
이 기능이 최종적으로 무엇을 남기는지 적는다.

예:
- `[텍스트 출력]`
- `[UI 표시]`
- `[컴포넌트 값 적용]`
- `[상태 갱신]`
- `[런타임 요약 생성]`

## 문서 버전 관리
- 현재 문서 버전: `1.0.0`
- 문서 상태: `Initial`
- 관리 원칙:
  - 이 문서는 한 번 작성하고 끝내는 문서가 아니라, 기능의 현재 상태가 바뀌면 함께 갱신한다.
  - 기능 설명 본문이 바뀌면 체인지로그도 같이 갱신한다.
  - 구현 변경 없이 표현만 다듬은 경우와, 기능 이해에 영향을 주는 내용 변경을 구분해서 기록한다.

### 버전 증가 기준
- `Major`
  - 기능 해석 자체가 바뀌는 수준의 대규모 재작성
  - 문서 범위가 다른 기능 묶음까지 확장되거나 재정의될 때
- `Minor`
  - 현재 기능 설명에 중요한 항목이 추가될 때
  - 새로운 표시 항목, 동작 조건, 연결 구조가 확인되어 본문 의미가 확장될 때
- `Patch`
  - 오탈자 수정
  - 표현 명확화
  - 근거 보강
  - 본문 의미는 유지한 채 설명 정밀도만 올라갈 때
