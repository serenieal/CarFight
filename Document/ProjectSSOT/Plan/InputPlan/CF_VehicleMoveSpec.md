# CF_VehicleMoveSpec

## 문서 목적
본 문서는 CarFight 차량 이동 입력 변경안의 **SSOT 명세서 v0.2**다.

이 문서는 특정 세션, 특정 AI, 특정 작업자에 의존하지 않고 다음 내용을 고정한다.

- 차량 이동 입력 해석 규칙
- 게임패드 2D 입력의 각도/강도 판정
- 검은 영역 유지 정책
- 후진 영역의 브레이크/후진 전환 규칙
- 장치 모드 허용/차단 규칙
- 하드코딩 금지 범위

---

## 1. 범위
본 명세는 차량 이동 입력 중 아래 항목만 다룬다.

- 전진
- 후진
- 브레이크
- 조향
- 장치 모드 기반 입력 허용/차단

본 명세는 아래 항목을 고정하지 않는다.

- 실제 물리 키 배치
- 실제 게임패드 물리 스틱 위치
- 핸드브레이크 버튼 배치
- 카메라 스틱 배치
- IMC 내부 데드존 값

위 항목은 **IMC(Input Mapping Context)** 또는 상위 입력 설정에서 결정한다.

---

## 2. 입력 해석 원칙

### 2-1. 배치와 해석의 분리
- 물리 입력 배치는 IMC가 담당한다.
- 차량 입력 로직은 입력 액션의 **최종 2D 값**만 해석한다.

### 2-2. 권장 이동 입력 액션
- 차량 이동용 입력 액션은 `Axis2D` 성격의 단일 이동 액션으로 통합하는 것을 기본안으로 한다.
- 예시 이름은 `IA_VehicleMove`지만, 실제 이름 고정은 필수가 아니다.

### 2-3. 논리 좌표계
입력 액션의 논리 좌표계는 아래처럼 해석한다.

- `+Y` : 전진 방향
- `-Y` : 후진 방향
- `+X` : 우향
- `-X` : 좌향

---

## 3. 각도 기준
입력 벡터 각도는 아래 기준으로 계산한다.

- 위 = `0°`
- 오른쪽 = `90°`
- 아래 = `180°`
- 왼쪽 = `270°`
- 시계 방향 증가
- 시작각 / 종료각 포함

즉, 비교 연산은 경계 포함 기준으로 처리한다.

---

## 4. 기본 영역 정의
기본 디폴트 값은 아래와 같다.

### 4-1. 쓰로틀 영역
- `260° ~ 100°`

### 4-2. 후진 영역
- `135° ~ 225°`

### 4-3. 검은 영역
- 위 두 영역을 제외한 나머지 전부

### 4-4. 조정 가능 여부
위 수치는 **조정 가능 변수**여야 한다.
즉, 네이티브 코드에 고정 상수로만 두지 않고 이후 튜닝 가능한 설정값으로 유지해야 한다.

---

## 5. 입력 강도 규칙
입력 강도는 입력 벡터의 반지름(`Magnitude`)으로 계산한다.

- 중심 = `0`
- 외곽 = `1`
- 중간값 = 선형

중심 데드존은 본 명세 범위에서 제외한다.
해당 값은 IMC 설정값을 신뢰한다.

---

## 6. 조향 규칙
조향은 영역 판정과 별개로 처리한다.

### 6-1. 기본 규칙
- `Steering = InputX`

### 6-2. 의도
- 검은 영역에서도 조향이 끊기지 않게 유지한다.
- 검은 영역의 역할을 가감속 방향 전환 완충에 집중시킨다.

---

## 7. 게임패드 영역별 해석

## 7-1. 쓰로틀 영역
입력 각도가 쓰로틀 영역이면 아래처럼 처리한다.

- `Throttle = Magnitude`
- `Brake = 0`
- `Steering = InputX`
- 최근 방향 의도 = `Forward`

---

## 7-2. 후진 영역
후진 영역은 차량 상태에 따라 다르게 해석한다.

### A. 차량이 전진 중일 때
- `Throttle = 0`
- `Brake = Magnitude`
- `Steering = InputX`
- 즉시 역추진하지 않는다.
- 이 단계는 **브레이크 의도**로 해석한다.

### B. 차량이 정지 근처 또는 정지 상태일 때
- 입력 해석 계층은 계속 `Brake = Magnitude`를 유지한다.
- `Steering = InputX`
- 후진 전환 자체는 **UE Chaos Vehicle의 `bUseAutoReverse=true` 경로**가 담당한다.
- 즉, CarFight 커스텀 입력 계층은 역기어를 직접 강제하지 않는다.
- 음수 스로틀을 직접 넣지 않는다.

### C. 구현 책임 분리
후진 관련 책임은 아래처럼 분리한다.

- CarFight 입력 해석 계층:
  - 각도 판정
  - 후진 의도 판정
  - 브레이크 전달
- Chaos Vehicle:
  - 브레이크 유지 중 정지 후 후진 전환
  - `bUseAutoReverse` 기반 reverse 처리

즉, **후진 영역의 의미는 유지하되, 실제 reverse 전환은 엔진 기본 기능이 최종 결정권을 가진다.**

---

## 7-3. 검은 영역
검은 영역은 무입력 구간이 아니라 **방향 전환 완충 구간**이다.

### 정책 이름
- `HoldDirection_UseCurrentMagnitude`

### 규칙
- 방향은 직전 유효 방향 의도를 유지한다.
- 강도는 현재 입력 반지름을 사용한다.
- 조향은 계속 유지한다.

### A. 직전 유효 방향이 전진이면
- `Throttle = Magnitude`
- `Brake = 0`
- `Steering = InputX`

### B. 직전 유효 방향이 후진이면
- 후진 의도를 유지한다.
- 강도는 현재 `Magnitude`를 사용한다.
- 실제 전달값은 **브레이크 유지**를 기본으로 한다.
- 실제 reverse 전환은 Chaos Vehicle의 `bUseAutoReverse`가 담당한다.
- `Steering = InputX`

### C. 최근 방향 의도가 없으면
- `Throttle = 0`
- `Brake = 0`
- `Steering = InputX`

즉, 게임 시작 직후처럼 방향 이력이 없는 상태에서는 검은 영역을 중립으로 해석한다.

---

## 8. 최근 방향 의도
검은 영역 처리를 위해 최근 방향 의도 상태를 유지해야 한다.

### 8-1. 상태값
- `None`
- `Forward`
- `Reverse`

### 8-2. 갱신 규칙
- 쓰로틀 영역 진입 시 `Forward`
- 후진 영역 또는 후진 유지 검은 영역에서 `Reverse`
- 키보드 전진 입력 시 `Forward`
- 키보드 후진 입력 시 `Reverse`

### 8-3. Fallback 규칙
최근 방향 의도가 `None`이면 속도로 보조 판정할 수 있다.

- 전방 속도 > 0 이면 `Forward`
- 전방 속도 < 0 이면 `Reverse`
- 전방 속도 = 0 이면 `None`

---

## 9. 키보드 규칙
키보드도 동일한 차량 이동 의미를 유지한다.

- 전진 입력
- 후진 입력
- 좌우 조향 입력

단, 본 명세는 `WASD`를 문서 의미상 예시로만 허용할 뿐 **물리 키를 고정하지 않는다.**
실제 매핑은 IMC에서 바꿀 수 있어야 한다.

권장안은 키보드도 가능하면 같은 `Axis2D` 이동 액션으로 합류시키는 것이다.
그 경우 키보드도 같은 해석 파이프를 사용한다.

---

## 10. 장치 모드 규칙

### 10-1. Auto
- 입력 장치를 고정하지 않는다.
- 현재 유효한 입력 액션 값을 그대로 해석한다.
- 키보드/게임패드 간 빠른 전환을 막지 않는다.

### 10-2. KeyboardMouseOnly
- 키보드/마우스 계열 입력만 허용한다.
- 게임패드 계열 입력은 무시한다.

### 10-3. GamepadOnly
- 게임패드 계열 입력만 허용한다.
- 키보드/마우스 계열 입력은 무시한다.

---

## 11. 하드코딩 금지 항목
아래는 본 기능 구현에서 하드코딩 금지 대상이다.

- 특정 물리 키 이름
- 특정 게임패드 버튼 이름
- 좌스틱 / 우스틱 고정
- 카메라 스틱 배치
- 핸드브레이크 버튼 배치
- IMC 데드존 수치
- 수동 역기어 강제
- 음수 스로틀 기반 후진 강제

---

## 12. 본 명세가 고정하는 항목
아래는 코드/설계에서 반드시 고정해야 한다.

- 2D 입력 벡터 해석 규칙
- 각도 기준계
- 영역 판정 방식
- 검은 영역 유지 정책
- 전진 중 후진 영역 = 브레이크 규칙
- 후진 전환은 `bUseAutoReverse` 기반으로 처리하는 구조
- 장치 모드별 허용/차단 규칙

---

## 13. 디폴트 튜닝값
- `ThrottleStartAngle = 260°`
- `ThrottleEndAngle = 100°`
- `ReverseStartAngle = 135°`
- `ReverseEndAngle = 225°`
- `AngleBoundaryInclusive = true`
- `MagnitudeResponse = Linear`
- `BlackZonePolicy = HoldDirection_UseCurrentMagnitude`
- `SteeringPolicy = UseInputX`
- `ReverseTransitionAuthority = ChaosVehicle_bUseAutoReverse`
