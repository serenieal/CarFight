# 문서 정제 체크리스트

## 반드시 남길 것

- 핵심 요구사항
- 제약사항
- 결정 이유
- 예외 조건
- 검증 기준
- 작업 맥락
- 초보자가 이해하는 데 필요한 최소 배경 설명

## 먼저 줄일 것

- 같은 뜻의 반복 문장
- 여러 번 다시 설명하는 정의
- 결론 없이 장황한 서론
- 실행 판단에 직접 도움이 되지 않는 장식성 표현
- 이미 다른 문장에서 충분히 설명된 예시

## 삭제 전 확인

- 이 문장이 없어도 요구사항 해석이 달라지지 않는가?
- 이 문장이 없어도 예외 상황 판단이 가능한가?
- 이 문장이 없어도 왜 이렇게 결정됐는지 이해 가능한가?
- 이 문장이 없어도 검증 기준을 잃지 않는가?
- 이 문장이 없어도 작업 맥락이 끊기지 않는가?
- 이 문장이 없어도 초보자가 따라올 수 있는가?

## 통합 시 확인

- 정말 같은 의미인지 확인했는가?
- 강한 문장 하나로 합쳤을 때 예외나 조건이 빠지지 않았는가?
- 통합 후 참조 흐름이 더 좋아졌는가?
- 통합 과정에서 결정 이유나 검증 기준이 사라지지 않았는가?

## 충돌 정보 확인

- 서로 충돌하는 설명을 임의로 하나만 남기지 않았는가?
- 최신 여부가 불분명한 항목을 조용히 삭제하지 않았는가?
- 필요하면 Open Question / Deprecated / Superseded / Note로 표시했는가?

## 구조 변경 확인

- 제목 또는 섹션 이동이 실제로 흐름을 개선했는가?
- 구조 변경이 최소 범위에서만 수행되었는가?
- 구조를 바꾸면서 문서의 원래 의도나 책임 경계가 흐려지지 않았는가?

## 최종 점검

- 문서가 짧아졌지만 모호해지지 않았는가?
- 정보 밀도와 가독성이 함께 좋아졌는가?
- 수정 범위가 최소한으로 유지되었는가?
- 삭제, 축약, 통합, 충돌 표기 내역을 Change Note에 남겼는가?
- 확인 필요 항목을 Change Note에 남겼는가?
- 사용자가 바로 검토할 수 있는 구조인가?
