# CF_SteerTest

## 문서 목적
본 문서는 `CF_SteerSpec.md` 기준 차량 조향 변경안의 PIE 검증 절차와 합격 기준을 정의한다.

---

## 1. 테스트 전 준비

### 1-1. 필수 준비
- 게임패드 `IA_VehicleMove` 신규 조작 경로 준비
- 키보드 기존 조작 경로 유지
- 차량 속도 / 조향값 / 입력값 디버그 확인 가능 상태
- `InputOwner` 디버그 확인 가능 상태

### 1-2. 권장 디버그 항목
- Raw Stick X/Y
- Stick Magnitude
- Stick Direction X/Y
- Target Steering
- Current Steering
- Steering Turn Rate
- Steering Return Rate
- Steering Returning To Center
- Vehicle Speed Kmh
- Forward Speed Kmh
- Input Owner

---

## 2. 기본 조향/가속 분리 검증

### ST-01. 작은 입력 + 최대 방향 조향
**목적**
스틱을 작게 기울여도 방향이 명확하면 목표 조향이 방향 기준으로 계산되는지 확인한다.

**절차**
1. 스틱을 오른쪽으로 작게 기울인다.
2. 스틱을 왼쪽으로 작게 기울인다.
3. `TargetSteering`을 확인한다.

**예상 결과**
- 입력 크기가 작아도 방향이 명확하면 목표 조향은 방향 기준으로 크게 잡힌다.
- 가속 / 브레이크 강도는 입력 크기에 따라 작게 유지된다.

### ST-02. 끝 입력과 작은 입력의 조향 목표 비교
**목적**
스틱을 살짝 오른쪽으로 기울인 경우와 끝까지 오른쪽으로 기울인 경우, 방향이 같으면 조향 목표가 같거나 유사한지 확인한다.

**예상 결과**
- 두 경우 모두 조향 목표는 방향 기준으로 계산된다.
- 입력 크기 차이는 가감속 강도에만 주로 반영된다.

### ST-03. 대각선 방향 조향
**목적**
대각선 입력에서 조향 목표가 방향 성분에 따라 자연스럽게 계산되는지 확인한다.

**예상 결과**
- 위오른쪽 대각선은 `TargetSteering`이 0과 1 사이 값이 된다.
- 순수 오른쪽 입력은 `TargetSteering`이 1에 가까워진다.

---

## 3. 조향 속도 제한 검증

### ST-04. 왼쪽 끝에서 오른쪽 끝 전환
**목적**
조향 목표가 급변해도 실제 조향값이 순간이동하지 않는지 확인한다.

**절차**
1. 왼쪽 방향으로 조향한다.
2. 스틱을 빠르게 오른쪽으로 넘긴다.
3. `CurrentSteering` 변화 속도를 확인한다.

**예상 결과**
- `TargetSteering`은 빠르게 바뀔 수 있다.
- `CurrentSteering`은 `SteeringLockToLockTimeSec` 기준으로 제한된 속도로 이동한다.

### ST-05. 조향값 튐 방지
**목적**
작은 입력 노이즈로 조향이 크게 튀지 않는지 확인한다.

**절차**
1. 스틱을 거의 중앙 근처에서 미세하게 움직인다.
2. `TargetSteering`과 `CurrentSteering`을 확인한다.

**예상 결과**
- `SteeringDirectionMinMagnitude` 이하에서는 조향 의도가 없는 것으로 처리된다.
- 작은 노이즈로 조향이 최대값으로 튀지 않는다.

---

## 4. 중립 복귀 검증

### ST-06. 스틱 중립 시 즉시 0 리셋 금지
**목적**
스틱을 놓았을 때 바퀴가 즉시 중립으로 순간이동하지 않는지 확인한다.

**절차**
1. 한쪽으로 조향한다.
2. 스틱을 중립으로 놓는다.
3. `CurrentSteering`이 0으로 이동하는 과정을 확인한다.

**예상 결과**
- `TargetSteering = 0.0`
- `CurrentSteering`은 천천히 0으로 이동한다.

### ST-07. 저속 중립 복귀
**목적**
저속에서 바퀴가 느리게 중립 복귀하는지 확인한다.

**절차**
1. 저속으로 이동한다.
2. 조향 후 스틱을 중립으로 놓는다.
3. `SteeringReturnRate`를 확인한다.

**예상 결과**
- 중립 복귀가 느리다.
- `SteeringReturnRate`가 낮게 계산된다.

### ST-08. 고속 중립 복귀
**목적**
고속에서 바퀴가 더 빠르게 중립 복귀하는지 확인한다.

**절차**
1. 중속 또는 고속으로 이동한다.
2. 조향 후 스틱을 중립으로 놓는다.
3. `SteeringReturnRate`를 확인한다.

**예상 결과**
- 저속보다 빠르게 중립 복귀한다.
- `SteeringReturnRate`가 높게 계산된다.

---

## 5. 기존 시스템 회귀 검증

### ST-09. 후진 전환 회귀
**목적**
조향 변경 후에도 `bUseAutoReverse` 기반 후진이 깨지지 않았는지 확인한다.

**예상 결과**
- 전진 중 뒤 입력은 브레이크로 동작한다.
- 정지 후 후진 전환이 정상 동작한다.

### ST-10. Runtime Input Ownership 회귀
**목적**
입력 소유권 전환이 조향 보간과 충돌하지 않는지 확인한다.

**예상 결과**
- 게임패드 신규 조작 사용 시 `InputOwner = VehicleMove2D`
- 키보드 기존 조작 사용 시 `InputOwner = LegacyAxis`
- release 시 다른 입력 경로의 조향값을 부정확하게 덮어쓰지 않는다.

### ST-11. 키보드 기존 조작 회귀
**목적**
키보드 기존 조작이 의도치 않게 바뀌지 않았는지 확인한다.

**예상 결과**
- 키보드 전진 / 브레이크 / 조향이 기존 방식으로 정상 동작한다.

---

## 6. 합격 기준
아래가 모두 만족되면 1차 합격으로 본다.

- 스틱 크기와 조향량의 직접 비례 관계가 제거된다.
- 스틱 크기는 가감속 강도에 반영된다.
- 스틱 방향은 목표 조향에 반영된다.
- 실제 조향값은 목표값으로 순간이동하지 않는다.
- 스틱 중립 시 실제 조향값은 속도 기반으로 0을 향해 복귀한다.
- 저속보다 고속에서 중립 복귀가 빠르다.
- 후진 전환과 입력 소유권 로직이 회귀하지 않는다.
